Safeguard Types

1. Confidentiality Complete

Objective: Ensure that sensitive information is accessible only to authorized entities.

Common Threats

  • Unauthorized access (hacking, phishing)
  • Data leaks or eavesdropping
  • Unencrypted communications

Countermeasures

  • Encryption: AES, TLS for data at rest/transit
  • Access Controls: RBAC, Principle of Least Privilege
  • Data Masking: Hide sensitive info in testing
  • Network Segmentation: Separate critical systems
✓

2. Integrity Complete

Objective: Ensure information remains accurate, complete, and unaltered.

Common Threats

  • Data manipulation (malicious edits)
  • Transmission errors (corruption)
  • Unauthorized modifications

Countermeasures

  • Cryptographic Hashing: SHA-256 verification
  • Digital Signatures: Authenticate source
  • Audit Trails: Detailed logs & monitoring
  • Redundancy: RAID configurations
⚡

3. Availability Complete

Objective: Ensure systems and data are accessible when needed.

Common Threats

  • DoS/DDoS attacks
  • System failures (hardware issues)
  • Natural disasters

Countermeasures

  • Redundancy: Backup servers, load balancers
  • DDoS Mitigation: Traffic filtering
  • Regular Backups: Geographically distributed
  • Disaster Recovery: Tested recovery strategies
ID

4. Authentication Complete

Objective: Ensure users and systems are who they claim to be.

Common Threats

  • Impersonation attacks
  • Stolen credentials
  • Weak or compromised passwords

Countermeasures

  • Multi-Factor Authentication (MFA): Password + biometrics
  • Strong Password Policies: Complex & rotation
  • Biometric Authentication: Fingerprints, facial recognition
  • Session Management: Secure cookies, timeouts
✎

5. Non-repudiation Complete

Objective: Ensure individuals cannot deny actions or transactions they performed.

Common Threats

  • Users denying sent data
  • Fraudulent denial of transactions
  • Repudiation of message sending

Countermeasures

  • Digital Signatures: Bind identity to message
  • Public Key Infrastructure (PKI): Certificates & keys
  • Transaction Logs: Tamper-proof records
  • Time Stamping: Trusted third-party validation
🛡

Integrated Countermeasures

Many threats target multiple aspects simultaneously (e.g., ransomware attacks Confidentiality, Integrity, and Availability).

Examples

  • IDPS: Monitor network traffic for suspicious activities
  • Zero Trust Architecture: Assume no trust by default
  • Regular Security Audits: Periodic evaluations

The House Analogy

Understanding security through real-world comparison

🔒

Confidentiality

Like having locks, curtains, and security systems to ensure only authorized people (homeowners) can enter or see inside. Prevents snooping by outsiders.

Example: Burglar alarm ensures unauthorized entry is detected
🏗

Integrity

Like ensuring the house structure (walls, roof, plumbing) is well-maintained and nothing inside (furniture, documents) is tampered with or damaged.

Example: Checking that important family documents remain unchanged
🔑

Availability

Ensures the house and amenities (water, electricity, internet) are always accessible when needed. You can always enter when necessary.

Example: Spare key ensures access even if main key is lost
